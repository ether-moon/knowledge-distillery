# Decision: 도메인 이름은 트리거 품질을 우선한다

**Decision**: 도메인 이름은 짧음보다 트리거 품질과 경계 명확성을 우선하며, 필요하면 더 길더라도 맥락이 충분히 드러나는 자기 설명적인 이름을 사용한다.

**Context**: Context Gate에서 도메인 목록을 경량 인덱스로 preload해 LLM이 탐색 네비게이션에 활용하는 방향을 검토했다. 이 방식에서는 본문 규칙보다 도메인 ID 자체가 탐색 품질을 크게 좌우하므로, 짧지만 모호한 이름은 라우팅 정확도를 떨어뜨린다.

**Rationale**: 도메인 이름은 저장용 식별자가 아니라 에이전트가 언제 해당 도메인을 조회해야 하는지 추론하는 트리거 인터페이스다. 따라서 `core`, `system`, `processing` 같은 짧고 범용적인 이름보다, `batch-refinement`, `vault-schema`, `plugin-runtime`처럼 책임과 경계를 드러내는 이름이 더 적합하다. 이름만으로도 적용 범위와 인접 비적용 범위를 어느 정도 추론할 수 있어야 하며, 이 요구를 만족하기 위해서는 다소 길더라도 자기 설명적인 이름을 허용하는 편이 전체 사용 품질에 유리하다.
